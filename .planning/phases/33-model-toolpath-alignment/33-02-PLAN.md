Create a pure utility that validates whether G-code cutting points lie near the transformed mesh surface.

**Create src/core/carve/alignment_validator.h:**

```cpp
#pragma once

#include "../gcode/gcode_types.h"
#include "../mesh/mesh.h"
#include "../types.h"
#include "model_fitter.h"

namespace dw {
namespace carve {

struct AlignmentResult {
    bool valid = false;      // Enough data to validate
    bool aligned = false;    // Points are near the mesh surface
    int pointsTested = 0;    // Number of G-code points sampled
    int pointsNear = 0;      // Number that are near the mesh surface
    f32 nearRatio = 0.0f;    // pointsNear / pointsTested
};

// Validate alignment between G-code toolpath and transformed mesh.
// Samples approximately samplePercent of cutting segments from the G-code,
// transforms mesh vertices using ModelFitter + FitParams, and checks
// whether sampled G-code points lie within toleranceMm of the mesh surface.
//
// Works entirely in G-code space (Z-up). Does NOT apply Y<->Z swap.
AlignmentResult validateAlignment(
    const gcode::Program& program,
    const Mesh& mesh,
    const ModelFitter& fitter,
    const FitParams& params,
    f32 samplePercent = 1.0f,     // % of cutting segments to test
    f32 toleranceMm = 1.0f);       // proximity threshold in mm

} // namespace carve
} // namespace dw
```

**Create src/core/carve/alignment_validator.cpp:**

Implement `validateAlignment()`:

1. **Early exit:** If `program.path.empty()` or `mesh.triangleCount() == 0`, return `{false, false, 0, 0, 0.0f}`.

2. **Forward-transform mesh vertices** into G-code space:
   - Create a `std::vector<Vec3> transformedVerts(mesh.vertexCount())`.
   - For each vertex, call `fitter.transform(vertex.position, params)` to get the G-code-space position. Note: ModelFitter::transform requires that `fitter.setModelBounds()` and `fitter.setStock()` have been called. The caller (ViewportPanel) ensures this. To avoid modifying the passed-in fitter, make a local copy: `ModelFitter localFitter = fitter;` -- but actually ModelFitter::transform is const, so we just need the fitter to have correct state. The caller constructs a temporary fitter with the right bounds/stock for each call.
   - Actually, simpler approach: the caller will pass a correctly configured ModelFitter, so just use it directly.

3. **Sample cutting G-code points:**
   - Collect indices of non-rapid segments: `for (size_t i = 0; i < program.path.size(); ++i) if (!program.path[i].isRapid) cuttingIndices.push_back(i);`
   - If `cuttingIndices.empty()`, return `{false, false, 0, 0, 0.0f}`.
   - Compute sample count: `int sampleCount = std::max(1, static_cast<int>(cuttingIndices.size() * samplePercent / 100.0f));`
   - Use deterministic sampling (stride-based, not random): `int stride = std::max(1, static_cast<int>(cuttingIndices.size()) / sampleCount);`
   - For each sampled segment, use the midpoint `(start + end) / 2` as the test point.

4. **Point-to-mesh proximity test (brute force):**
   - For each sampled G-code point, find the minimum distance to any triangle in the transformed mesh.
   - Use point-to-triangle distance: for each triangle (3 vertices from transformedVerts via mesh indices), compute the closest point on the triangle to the test point, then compute the distance.
   - Point-to-triangle closest point algorithm:
     ```
     // Given triangle vertices a,b,c and point p:
     // Project p onto triangle plane
     // Check if projection is inside triangle (barycentric coordinates)
     // If inside: distance = |p - projection|
     // If outside: distance = min distance to each edge
     ```
   - Use the standard Ericson/Eberly point-triangle distance algorithm. Implement as a local helper function `closestPointOnTriangle(Vec3 p, Vec3 a, Vec3 b, Vec3 c) -> Vec3`.
   - If the minimum distance to any triangle is < toleranceMm, count the point as "near".

5. **Compute result:**
   - `nearRatio = pointsNear / pointsTested`
   - `aligned = nearRatio >= 0.7f` (70% threshold -- most cutting points should be near the surface, but some may be on approach/retract portions)
   - `valid = true`

**Performance note:** With ~1% sampling and brute-force triangle iteration, this handles typical meshes (10K-100K triangles) and toolpaths (1K-50K segments) in under 100ms. No spatial acceleration needed for this use case.
