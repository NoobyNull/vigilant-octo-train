<objective>
Add G-code toolpath line rendering capability to ViewportPanel using the existing shared camera.

Purpose: This is the core rendering infrastructure for the unified viewport. ViewportPanel currently renders triangle meshes (STL/OBJ/3MF) and toolpath meshes (from Direct Carve). It needs to also render G-code line geometry (GL_LINES) with height-based depth coloring, porting the rendering logic from GCodePanel's renderPathView into the unified viewport.

Output: ViewportPanel can build and render G-code line geometry from a gcode::Program, with height-colored cutting paths and flat-colored rapids/plunges/retracts, all using its existing camera and framebuffer.
</objective>

<task type="auto">
  <name>Task 2: Add G-code line geometry infrastructure and rendering to ViewportPanel</name>
  <files>src/ui/panels/viewport_panel.h, src/ui/panels/viewport_panel.cpp</files>
  <action>
Port G-code line rendering from GCodePanel into ViewportPanel. This is the bulk of the plan.

**In viewport_panel.h:**

Add `#include "../../core/gcode/gcode_types.h"` at the top.

Add the following public methods:
- `void setGCodeProgram(const gcode::Program& program);` — receives parsed G-code data, stores it, marks geometry dirty.
- `void clearGCodeProgram();` — clears G-code data and destroys line geometry.
- `bool hasGCode() const { return !m_gcodeProgram.path.empty(); }` — query whether G-code is loaded.

Add the following private members (G-code line geometry section, clearly commented):
```cpp
// --- G-code line rendering ---
gcode::Program m_gcodeProgram;
GLuint m_gcodeVAO = 0;
GLuint m_gcodeVBO = 0;
u32 m_gcRapidStart = 0, m_gcRapidCount = 0;
u32 m_gcCutStart = 0, m_gcCutCount = 0;
u32 m_gcPlungeStart = 0, m_gcPlungeCount = 0;
u32 m_gcRetractStart = 0, m_gcRetractCount = 0;
bool m_gcodeDirty = false;
```

Add private methods:
- `void buildGCodeGeometry();` — builds the GL_LINES VBO from m_gcodeProgram.path.
- `void destroyGCodeGeometry();` — deletes VAO/VBO, resets counts.
- `void renderGCodeLines();` — draws the line geometry using flat + height-line shaders.

**In viewport_panel.cpp:**

Add `#include "../../render/gl_utils.h"` for the GL_CHECK macro.

**Implement `setGCodeProgram()`:**
- Store the program: `m_gcodeProgram = program;`
- Mark dirty: `m_gcodeDirty = true;`
- If no mesh is loaded (`!m_mesh`), fit camera to the G-code bounds (swap Y<->Z for renderer convention): `Vec3 bMin{program.boundsMin.x, program.boundsMin.z, program.boundsMin.y}; Vec3 bMax{program.boundsMax.x, program.boundsMax.z, program.boundsMax.y}; m_camera.fitToBounds(bMin, bMax);`
- Invalidate ViewCube cache.

**Implement `clearGCodeProgram()`:**
- `m_gcodeProgram = gcode::Program{};`
- Call `destroyGCodeGeometry();`

**Implement `destroyGCodeGeometry()`:**
- Delete m_gcodeVBO and m_gcodeVAO if non-zero (matching GCodePanel::destroyPathGeometry pattern).
- Reset all count/start members to 0.

**Implement `buildGCodeGeometry()`:**
Port from GCodePanel::buildPathGeometry() but ONLY the standard mode (NOT color-by-tool, NOT simulation overlay — those come in Phase 32 and 34 respectively). Specifically:

1. Call `destroyGCodeGeometry();`
2. If `m_gcodeProgram.path.empty()`, set dirty=false and return.
3. Use the same Y<->Z swap lambda: `addSegVerts` pushes `(seg.start.x, seg.start.z, seg.start.y, seg.end.x, seg.end.z, seg.end.y)`.
4. Classify segments into 4 vectors: rapidVerts, cutVerts, plungeVerts, retractVerts. Use the same Z-dominant heuristic from GCodePanel (dz*dz > dxy2 * 0.25f). For this phase, show ALL move types (no filter booleans yet — those come in Phase 32).
5. Concatenate into allVerts and record start/count for each group (same logic as GCodePanel lines 1157-1171).
6. Upload to GPU: glGenVertexArrays, glGenBuffers, glBufferData, glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3*sizeof(f32), nullptr), glEnableVertexAttribArray(0). Use GL_CHECK macro.
7. Set `m_gcodeDirty = false;`

**Implement `renderGCodeLines()`:**
Port the standard-mode rendering from GCodePanel::renderPathView() lines 1386-1427:

1. If `m_gcodeVAO == 0` return.
2. Get `Shader& flat = m_renderer.flatShader();` and `Shader& heightShader = m_renderer.heightLineShader();`
3. Compute `Mat4 mvp = m_camera.viewProjectionMatrix();`
4. Compute Y bounds in renderer space: `float yMin = m_gcodeProgram.boundsMin.z; float yMax = m_gcodeProgram.boundsMax.z;` (Z in G-code maps to Y in renderer).
5. `glDisable(GL_CULL_FACE);` and `glLineWidth(1.5f);`
6. `glBindVertexArray(m_gcodeVAO);`
7. Draw rapids with flat shader: gray `Vec4{0.4f, 0.4f, 0.4f, 0.5f}`
8. Draw cuts with height-line shader: `uColorLow = Vec4{0.05f, 0.15f, 0.5f, 1.0f}`, `uColorHigh = Vec4{0.3f, 0.8f, 1.0f, 1.0f}`
9. Draw plunges with flat shader: orange `Vec4{1.0f, 0.5f, 0.1f, 1.0f}`
10. Draw retracts with flat shader: green `Vec4{0.3f, 0.8f, 0.3f, 0.6f}`
11. `glBindVertexArray(0);` and `glEnable(GL_CULL_FACE);`

**Integrate into renderViewport():**
In the `renderViewport()` method, AFTER the existing toolpath rendering block (`if (m_gpuToolpath.vao != 0) { ... }`) and BEFORE the CNC live tool position block (`if (m_cncConnected) { ... }`):

1. Add: `if (m_gcodeDirty) buildGCodeGeometry();`
2. Add: `renderGCodeLines();`

**IMPORTANT NOTES:**
- Do NOT add filter toggles (m_showRapid, etc.) — those come in Phase 32. For now, render ALL move types unconditionally.
- Do NOT add color-by-tool mode — that comes in Phase 32.
- Do NOT add simulation overlay — that comes in Phase 34.
- The existing m_gpuToolpath (triangle-based toolpath mesh from Direct Carve) remains untouched. G-code lines are a separate rendering layer.
- The coordinate swap (Y<->Z) is critical: G-code uses Z-up, the renderer uses Y-up. This swap happens during geometry building, not during rendering.
