Wire G-code file loading so parsed programs reach ViewportPanel for rendering.

Purpose: Plan 01 added setGCodeProgram() to ViewportPanel but nothing calls it. This plan adds the callback plumbing so that when GCodePanel loads a .nc file (from any entry point — Open button, recent files, project panel), the parsed gcode::Program is forwarded to ViewportPanel. External .nc files then render in the viewport standalone, satisfying ALN-03.

Output: Loading a G-code file in GCodePanel automatically displays the toolpath in the main viewport. Clearing G-code in GCodePanel also clears the viewport overlay.

  <files>src/app/application_wiring.cpp, src/app/application_wiring_cnc.cpp</files>
  <action>
Wire the new GCodePanel callbacks to forward G-code programs to ViewportPanel.

Find the section in application_wiring.cpp or application_wiring_cnc.cpp where GCodePanel dependencies are set up (likely near `gcp->setGCodeRepository(...)` in application_wiring_cnc.cpp around line 75-80).

After the existing GCodePanel setup, add the callback wiring:

```cpp
gcp->setOnProgramLoaded([this](const gcode::Program& prog) {
    if (auto* vp = m_uiManager->viewportPanel())
        vp->setGCodeProgram(prog);
});

gcp->setOnProgramCleared([this]() {
    if (auto* vp = m_uiManager->viewportPanel())
        vp->clearGCodeProgram();
});
```

This ensures:
- When GCodePanel loads a .nc file (from Open button, recent files, or project panel), the program is forwarded to ViewportPanel.
- When GCodePanel clears (e.g., user closes the file), ViewportPanel also clears its G-code overlay.
- The null check on viewportPanel() prevents crashes if panel ordering changes.

Verify the include for gcode_types.h is accessible from the wiring file (it should be, since GCodePanel's header already includes it).
